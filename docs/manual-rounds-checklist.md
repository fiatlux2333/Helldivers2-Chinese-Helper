# ✅ 手动验收速查清单

配套 [`windows-probe.md`](windows-probe.md)（T01–T10、F01–F12）。把「记事本 / 误投 / 输入法」三轮映射到代码与判定标准，供真机速查。

> **前置**：先完成 `windows-probe.md` 的系统准备与 A 层逻辑验证。任一项「必测」失败 → 停，先修。

## 🔌 环境判定（本仓库现状）

| 项 | 值 |
| --- | --- |
| 标题关键词 | 默认 `HELLDIVERS`（`src-tauri/src/core/config.rs`） |
| 记事本探针文件名 | `HELLDIVERS-input-test.txt` |
| 字符上限 / 批大小 / 延迟 | `100` / `5` / `50ms` |
| 填入事件 | 仅 `KEYEVENTF_UNICODE` 按下+抬起，**从不** Enter/Esc |

## 📝 第一轮：记事本探针（必做）

**目标**：捕获 → 最小化 → 切回 → 锁定 → 填入；只填字、不自动回车。

**准备**

1. 打开记事本 → 另存为 `HELLDIVERS-input-test.txt`
2. 确认标题栏含 `HELLDIVERS`
3. 光标停在编辑区

| 步骤 | 操作 | 预期 | 结果 |
| --- | --- | --- | --- |
| 1 | 点「捕获 HD2」 | 助手最小化；约 4 秒内切回记事本 | ☐ |
| 2 | 助手重新出现 | 提示目标已锁定；`target.valid === true` | ☐ |
| 3 | 输入 `你好，绝地潜兵` 并填入 | 记事本出现**完整**文字；无自动回车 | ☐ |
| 4 | 复查 | 不漏字 / 不重复 / 顺序正确 / 无控制字符 / 无 Enter/Esc | ☐ |

**必测文本**

- [ ] `你好，绝地潜兵`
- [ ] `A点集合，30秒`
- [ ] `测试𠮷字符`（非 BMP，代理对不得拆开）
- [ ] `中文测试🙂继续`
- [ ] `这是一条用于验证多批Unicode输入是否完整的测试消息`

## 🛡 第二轮：误投与失败保护（必做）

依据：`inject_probe_text` 发送前 `restore_foreground` + `validate_foreground`；`injector` 每批再校验，变化即停并置 `partial_prefix_possible`。

- [ ] **A. 填入时切走** → 浏览器**无**文字；助手报目标变化（`TargetChanged`）
- [ ] **B. 捕获后关记事本** → **拒绝发送**，不写别的窗
- [ ] **C. 多批中途切窗** → 中途停止，提示可能已有部分前缀；**不要直接重试整段**
- [ ] **D. 权限不一致**（管理日记事本 + 普通助手）→ 拒绝（`IntegrityIncompatible`）

## ⌨️ 第三轮：输入法

依据：`useCompositionLatch.ts` + `App.vue` 的 keydown/keyup。

- [ ] **微软拼音**：候选 Enter 不填入；组字结束再 Enter 才填入；候选 Esc 交还 IME；长按不重复；修饰键+Enter 不提交；候选上下键不走历史
- [ ] **搜狗**：同上
- [ ] **微信输入法**：同上

> 单元测试层已由 `useCompositionLatch.test.ts` 覆盖；真机做确认即可。

## 🎮 第四轮：HD2 + 热键（推荐）

- [ ] 窗口化 / 无边框，先开聊天框
- [ ] 捕获成功（含 `stingray_window` 进程名回退场景）
- [ ] 填入后游戏内**手动** Enter 发送
- [ ] 无需重捕获即可下一句
- [ ] 填入后助手保持最小化、不抢焦点
- [ ] 热键单击唤回；约 0.5s 内连按两次收起
- [ ] 热键**不**自动填字 / 不自动 Enter
