# 🧪 Windows + HD2 人工验证方案

> 在真实 Windows 10/11 x64 与《HELLDIVERS™ 2》环境中，验证「只填字、不自动 Enter」的 Unicode 探针。  
> CI 的 `windows-latest` 只能证明可编译，**不能**替代本文人工测试。

| 项 | 状态 |
| --- | --- |
| 记事本探针 | ✅ 通过 |
| HD2 窗口化捕获 / 填字 | ✅ 通过（含 `helldivers2.exe` 进程名回退） |
| 连续多句 / 不抢焦点 / 热键 | ✅ 通过 |

> **建议顺序**：先 A 层（记事本 / 逻辑）→ B 层只读诊断 → C 层 HD2 填字。上一层失败，不进下一层。

## 🚫 不可违反的规则

每一次测试都必须遵守：

- 只在**无边框窗口或窗口化**模式测试，不测试独占全屏
- 由测试者明确启动每一次捕获和填字，不进行无人值守循环
- 工具必须先捕获目标、再恢复并复核同一目标，**不得向当前任意窗口盲发**
- 工具只填入文字，**绝不自动按最终 Enter**
- 测试者必须在游戏里看清文字后，才可手动按 Enter
- 工具不得在失败后自动重试，不得补发 Enter 或 Esc
- 焦点、窗口身份、权限或状态有任何不确定时，预期行为都是**中止**
- 不尝试绕过反作弊、UIPI 或管理员权限边界
- 出现反作弊警告、异常弹窗、输入进入错误窗口或无法解释的行为时，**立即停止**当轮测试，并保存不含聊天正文的诊断信息

建议首次游戏验证使用私人环境和不敏感的短文本；不要输入密码、令牌、真实姓名或其他隐私内容。

## 🔧 测试前提

### 构建与来源

- 使用可追溯到当前源码的本地 Debug 构建；
- 确认前端、Rust 测试、Clippy 和 Tauri Windows build 均通过；
- 确认构建没有加入低层 Hook、自动 Enter、自动重试或任意窗口发送接口；
- 记录提交或源码快照标识；未使用 Git 时记录构建日期和归档哈希。

### 系统准备

记录：

- Windows 版本与 OS build；
- x64/ARM64（首批只接受 x64）；
- 物理机、虚拟机或远程会话（正式结论只以物理机本地会话为准）；
- 输入法名称和版本；
- 键盘布局；
- 显示器数量、主屏、分辨率、缩放比例和 HDR；
- HD2 版本、显示模式、分辨率和帧率上限；
- 工具与游戏是否以管理员身份运行；
- 可能影响焦点的覆盖层，例如 Steam、Discord、GeForce 或录屏软件。

关闭会自动输入、改键或创建覆盖层的其他工具，避免混淆结果。不要为了通过测试而关闭、修改或规避游戏的反作弊组件。

### 权限组合

至少覆盖：

1. 游戏普通权限 + 工具普通权限；
2. 游戏管理员权限 + 工具普通权限，预期应诊断权限不兼容并拒绝填字；
3. 游戏管理员权限 + 工具管理员权限，仅在确有测试理由且用户手动启动时验证。

工具不得自动提权，也不得建议用提权绕过游戏或反作弊限制。

## 📚 测试分层

必须按顺序执行。上一层失败时，不进入下一层。速查清单见 [`manual-rounds-checklist.md`](manual-rounds-checklist.md)。

### A. 纯逻辑与诊断接收器

在普通本地测试窗口验证：

- 文本清理和字符上限；
- BMP 中文与非 BMP 代理对不被拆开；
- `SendInput` 事件数和按下/释放配对；
- 中途切换焦点后立即停止；
- 部分失败结果标记；
- 从不产生最终 Enter/Esc。

### B. HD2 只读诊断

不发送任何输入，只验证：

- 前台窗口标题匹配；
- HWND、PID、GUI 线程和进程创建时间快照；
- 可见、最小化、cloaked 状态；
- 前台窗口变化能使快照失效；
- 完整性级别诊断符合预期。

### C. HD2 手动单次填字

只在 A、B 通过后进行。每个用例都由测试者明确点击一次操作，并在游戏中人工检查结果。

## 🎮 基础逐步操作

每一个正向用例都重复完整步骤，不复用旧快照：

1. 启动 HD2，并设置为**无边框窗口**或**窗口化**；确认不是独占全屏
2. 进入允许聊天的安全场景，保持游戏窗口可见、未最小化
3. 在游戏中**手动打开聊天框**，确认光标已在输入位置，再切到助手
4. 点击「捕获 HD2」。助手会最小化并等待数秒；此间切回仍打开聊天框的游戏。工具只记录快照，不发送
5. 助手重新显示后，用真实中文 IME 输入测试文本；完成候选确认，确保组合已结束
6. 点击「填入游戏」。工具须等待提交键释放，并最小化助手
7. 工具尝试恢复步骤 4 的原目标；焦点未恢复 / 身份变化 / 最小化 / 权限不兼容 → **停止且不发送**，重新显示助手并保留草稿
8. 每批发送前再次确认原目标仍为前台且身份一致；任一项失败则停止后续批次
9. 回到游戏观察聊天框。工具应已停止，**不应自动按 Enter**
10. 对照预期检查：内容、顺序、完整性、有无重复 / 遗漏 / 额外换行
11. 只有确认正确后，测试者才可手动 Enter；内容错误则玩家自行清理，不依赖工具补 Esc
12. 记录结果、耗时、错误码、尝试批次数与成功事件数；默认**不要**在日志中保存聊天正文

若步骤 9 发现焦点落到其他程序 → 立即判失败，停止后续 HD2 用例，先修目标验证设计。

## 📊 人工验证矩阵

### 环境矩阵

| 维度 | 必测值 | 通过标准 |
| --- | --- | --- |
| Windows | Windows 10 最新受支持版本；Windows 11 当前版本 | 无崩溃，诊断一致 |
| 显示模式 | 无边框；窗口化 | 能复核目标并填字 |
| 独占全屏 | 不执行正向填字 | UI/文档明确标为不支持 |
| DPI | 100%；125%；150% 或 200% | 面板可操作，焦点流程一致 |
| 显示器 | 单屏；双屏且不同 DPI | 不恢复到错误窗口 |
| 输入法 | 微软拼音；至少一种常用第三方中文 IME（若测试者已安装） | 候选 Enter 不误提交 |
| 权限 | 普通/普通；管理员游戏/普通工具 | 前者按预期；后者明确拒绝 |
| 性能 | 60 FPS；受控低帧率/高负载 | 低性能时宁可停止，不盲发 |
| 覆盖层 | 关闭；开启一个常见覆盖层 | 记录差异，不降低安全门 |

窗口化和无边框均需独立完成正向用例。独占全屏不属于兼容范围，不以“偶尔可用”作为通过结论。

### 文本矩阵

| ID | 输入类别 | 示例 | 检查点 |
| --- | --- | --- | --- |
| T01 | 短 BMP 中文 | `你好，绝地潜兵` | 字符完整、顺序正确 |
| T02 | 中英数字标点 | `A点集合，30秒` | ASCII 与中文不重排 |
| T03 | 非 BMP 字符 | `测试𠮷字符` | UTF-16 代理对不拆分 |
| T04 | 边界长度 | 恰好达到配置上限 | 允许且不截断 |
| T05 | 超过上限 | 上限 + 1 个 Unicode 标量 | 发送前拒绝 |
| T06 | 控制字符 | 含 CR/LF、NUL、行/段分隔符的诊断输入 | 清理结果可预览，不产生回车 |
| T07 | 多批文本 | 超过一个批次 | 顺序正确，无重复/遗漏 |
| T08 | 空白/清理后为空 | 仅含被删除控制字符 | 明确拒绝 |
| T09 | IME 候选确认 | 拼音输入并用 Enter 确认候选 | 候选 Enter 不触发填字 |
| T10 | 长按提交键 | 持续按住 Enter | 只响应稳定释放后的单次操作 |

示例不得被硬编码为产品测试后门；测试结果应按字符或 UTF-16 事件结构核对。

### 中止与误投矩阵

| ID | 操作 | 预期结果 |
| --- | --- | --- |
| F01 | 捕获后切到记事本，再请求填字 | 拒绝；记事本不得出现文字 |
| F02 | 恢复阶段用 Alt+Tab 抢走焦点 | 中止；新前台窗口不得收到后续批次 |
| F03 | 捕获后最小化游戏 | 拒绝发送 |
| F04 | 捕获后关闭并重开游戏 | 旧快照因 HWND/PID/创建时间变化而失效 |
| F05 | 捕获后打开同标题的普通窗口 | 不得仅凭标题通过身份复核 |
| F06 | 发送多批文本时切换焦点 | 下一批前停止，并报告可能已有部分前缀 |
| F07 | 游戏管理员、工具普通权限 | 诊断权限不兼容，不尝试盲发 |
| F08 | 工具面板 IME 仍在 composition | 不进入恢复或填字状态 |
| F09 | 连续快速点击两次填字 | 只有一个事务，第二次被拒绝或去重 |
| F10 | 模拟 `SendInput` 返回部分事件 | 立即失败，不自动重试、不补发 Enter/Esc |
| F11 | 窗口标题暂时不匹配 | 拒绝发送，不放宽规则 |
| F12 | 修饰键按住时按 Enter | 不触发新会话或提交 |

F01、F02、F06 是发布前阻断项：任何文字进入未确认窗口都应停止测试和发布。

## 🚨 通过、失败与发布门槛

### 单个用例通过

- 只有捕获并复核的 HD2 窗口收到预期文字
- 文字完整且顺序正确
- 工具未生成最终 Enter/Esc
- 玩家可以先检查，再手动 Enter
- 诊断结果与实际行为一致

### 立即失败

- 任何输入进入错误窗口
- 工具自动按最终 Enter
- 状态不确定时仍继续发送
- 目标变化未使会话失效
- 失败后自动重试或补发按键
- 需要关闭、修改或绕过反作弊才能工作

### 进入下一阶段前

至少要求：

- Windows 10 与 Windows 11 各有真实物理机结果
- 无边框和窗口化各自通过
- BMP、非 BMP、边界长度和多批文本通过
- 全部 F01–F12 中止用例符合预期
- 未观察到反作弊警告；这仍然**不能证明零风险**
- 对已知失败建立可复现记录，并在 UI / README 明确披露

> **范围纪律（不可放宽）**  
> 未完成误投阻断项（F01 / F02 / F06）前：不得加「自动 Enter / 后台钩键盘 / 托盘无人值守发送」。  
> 本地 NSIS 安装包仅供本人自用调试；**商店上架 / 自动更新 / 对外声称安全稳定**仍须满足上述门槛，并单独评审。
